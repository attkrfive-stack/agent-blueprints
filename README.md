# Agent Blueprints & Components

**Purpose:**
This repository is your personal library for building capable agent systems by intelligently composing battle-tested components from existing high-quality work.

**Core Philosophy:**
- Don’t reinvent the wheel
- Pull proven, well-designed pieces
- Document the "why" behind every choice
- Create reusable blueprints you can compose later

**Goal:**
Reduce overwhelm by having one clean, structured place to organize ideas, components, and blueprints instead of scattered starred repositories.

---

## Repository Structure

```
agent-blueprints/
┌── blueprints/           # High-level agent designs & architectures
│   ├── memory-agents/
│   ├── tool-use-agents/
│   └── multi-agent-systems/
┌── components/           # Reusable building blocks
│   ├── memory/             # Memory systems & architectures
│   ├── tools/              # Individual tools & capabilities
│   ├── orchestration/      # Multi-agent coordination patterns
│   └── data/               # Data ingestion & processing
┌── docs/                 # Reasoning, notes, and documentation
│   ├── why/                # Why specific components were chosen
│   └── blueprints/         # Blueprint explanations
┌── references/           # Links & notes from starred repos
└── templates/            # Reusable templates and starting points
```

---

## How to Use This Repo

1. **Explore** your starred repositories
2. **Pull** useful components or patterns into the appropriate folder
3. **Document** the "why" in `docs/why/`
4. **Build blueprints** in the `blueprints/` folder when you have a clear vision
5. **Compose** later by combining components into new agent systems

**Rule of thumb:** If something feels useful for building agents, it belongs here with context.
